Cayley Graph

Visual encoding of a group's structure — vertices are elements, edges are generators
DEFINITION

Given a group G and a generating set S, the Cayley graph Γ(G,S) has one vertex per group element, with a directed edge from g to gs for each generator s∈S.

FORMAL DEFINITION
V = G, E = {(g, gs) : g∈G, s∈S}
KEY PROPERTIES
PropertyMeaning
Vertex-transitiveThe graph looks the same from every vertex
Degree = |S|Each vertex has exactly |S| outgoing edges
Connected iff S generates GS must span the whole group
DiameterMax steps to reach any element — the group's "width"
EXAMPLES
CYCLIC GROUP Z_n — GENERATORS {1}
A regular polygon with n vertices. One generator = one ring.
FREE GROUP F_2 — GENERATORS {a, b}
An infinite 4-regular tree. The Banach-Tarski decomposition lives here.
PTOLEMY CONNECTION
HYPERWEBSTER / GITНEXUS

The HyperWebster address space is navigated via a Cayley graph over the charset group. Each generator is a Unicode codepoint; each path through the graph is a Horner evaluation sequence. GitNexus (the planned semantic corpus visualizer) renders the HyperIndex as a Cayley-graph-style blockchain visualization — each commit a vertex, each semantic relationship an edge. The diameter of the charset Cayley graph bounds the worst-case address lookup depth.
