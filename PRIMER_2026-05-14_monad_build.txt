WHAT THE MONAD IS
========================================================================

The Monad is NOT:
  - A chatbot wrapper
  - A vector database
  - A retrieval system
  - An LLM of any kind
  - A transformer

The Monad IS:
  A self-deepening lexicographic database where:
  - Every word has a permanent address (a Riemann zero)
  - Addresses never change
  - What grows is the DEPTH at each address — the relational structure
    connecting neighboring zeros built up from context
  - Learning = deepening, not adding
  - Recall = traversing the depth structure from a query zero outward
  - Response = emitting words whose zeros are most deeply connected
    to the query zeros

The primitive unit is a RELATIONSHIP between zeros, not a word or token.

When the engine processes "the dog chased the cat":
  - Each word finds its zero
  - The zeros that fired together in the same context window
    get their co-activation weight increased
  - "dog" and "chase" become more deeply connected
  - Same zeros. Deeper connection.

This is how humans learn. "Dog" at age 2 and age 40 map to the same zero.
What changes is the depth of relational structure hanging off that zero.

THE CODE ARCHITECTURE

class Monad:
    
    # Core engine — the heartbeat
    engine: Understand          # from semantic_engine.py, tau configurable
    
    # The memory — grows with learning, never resets addresses
    # zero_index (int 1-N) → dict of {neighbor_zero_index: float weight}
    memory: dict[int, dict[int, float]]
    
    # DC state per zero — the depth
    # zero_index → current DC value (accumulated via Capacitor logic)
    depth: dict[int, float]
    
    def learn(self, text: str) -> None:
        """
        Pass text through engine word by word.
        Record which zeros co-activate in the same context window.
        Deepen co-activation weights.
        Context window: sentence boundary (split on . ? !)
        Within each sentence: all zero pairs get weight increment.
        Weight increment = product of their individual E values.
        Do NOT store the text. Store only the zero relationships.
        """
    
    def prompt(self, query: str) -> list[int]:
        """
        Process query words through engine.
        Return list of activated zero indices, sorted by DC depth descending.
        These are the starting nodes for traversal.
        """
    
    def respond(self, query: str, max_words: int = 50) -> str:
        """
        1. Get activated zeros from prompt(query)
        2. Traverse memory graph: from activated zeros, 
           follow highest-weight connections outward
        3. Collect zeros visited in traversal order
        4. For each visited zero, find the word in learned vocabulary
           whose zero index matches and whose E is closest to the zero's gamma/50
        5. Emit those words in traversal order
        Return: string of response words
        """
    
    def status(self) -> dict:
        """
        Return: total zeros activated, total relationships stored,
        deepest zero (highest DC), current tau
        """

IMPLEMENTATION NOTES
========================================================================

1. ZERO INDEX
   Use integer index into RIEMANN_ZEROS list (0-based or 1-based, your choice,
   be consistent). The gamma value is the canonical address — the index is
   just a convenience handle.

2. CONTEXT WINDOW
   Sentence boundary is correct. Split on ['.', '?', '!', '\n'].
   Within a sentence, every pair of words that land on DIFFERENT zeros
   gets a co-activation weight increase. Same-zero pairs deepen the DC
   at that zero via the Capacitor — they don't create a self-loop edge.

3. CO-ACTIVATION WEIGHT
   weight_increment = E_word1 * E_word2 * (1 / distance)
   where distance = abs(gamma1 - gamma2) in the zero spectrum.
   Nearby zeros that co-activate get stronger connections than distant ones.
   This is not a design choice — it follows from the spectral geometry.

4. VOCABULARY MAP
   Keep a simple dict: zero_index → list of (word, E) tuples
   Every word seen during learn() gets added here.
   respond() uses this to convert zeros back to words.

5. RESPONSE TRAVERSAL
   BFS from activated zeros, following memory edges by weight (highest first).
   Stop when max_words reached or graph exhausted.
   Emit the highest-E word at each visited zero.

6. TAU (Capacitor memory depth)
   Default tau=5.0 (from semantic engine session).
   Higher tau = longer memory = slower to update = more stable depth.
   Make it configurable at Monad init.

7. NO PERSISTENCE REQUIRED IN V1
   Memory lives in RAM. learn() builds it during the session.
   Persistence (save/load) is a v2 feature.

8. THE BAO / ERROR CHECK CONNECTION (DIRECTION — not required for v1)

   The DC sum of a complete Q+R exchange approaches Omega=0.56714.
