PTOLEMY IS A KERNEL
════════════════════════════════════════════════════════

Ptolemy is not an LLM. Not a transformer. Not a chatbot.
He is a kernel. Faces are kernel modules.
All inter-Face communication routes through the kernel bus.
No Face speaks directly to another.

PtolemyKernel (was PtolemyFace) owns:
  - The /DataInput bus
  - The thread pool
  - The monad pool (shared memory)
  - Both context buffer tiers
  - The ThalamicGate
  - The ExecutiveLayer goal state

════════════════════════════════════════════════════════
THREAD MODEL
════════════════════════════════════════════════════════

5 threads total.

Thread 0 — I/O (always alive, never parked)
           /DataInput bus → ThalamicGate → output cycle
           Heartbeat of the kernel.

Thread 1 — User directive
           Activates on user input.
           Immediately reclaims resources from autonomous threads.

Threads 2-4 — Autonomous (nice+10)
           Self-directed queries from recent context buffer.
           Parked (not killed) on "Full Attention Please."
           Each thread maintains a journal of current work.
           On release: resume from journal, not fresh start.

"Full Attention Please":
           Threads 2-4 finish current atomic operation.
           Then park. Weights never left in inconsistent state.
           Waiting for kernel to assign to user directive.

GPU sits outside thread model.
All threads share GPU via PyTorch CUDA stream manager.
GPU is hardware resource, not a thread.

════════════════════════════════════════════════════════
CUDA NOTE
════════════════════════════════════════════════════════

PyTorch .to('cuda') = GPU as parallel float processor.
All nn.Module operations get this for free.

CUDA kernels = custom C code running directly on GPU.
Bypasses PyTorch overhead entirely.
Sedenion multiplication is the first candidate for
Ptolemy++ CUDA kernel — 16x16 sign table,
256 multiply-accumulate ops, runs in single GPU warp.

Prototype: pure PyTorch sedenion implementation.
Production: Ptolemy++ CUDA kernel.

                             Not statistical co-occurrence.

  BrocaNetwork             — REPLACE ENTIRELY with:
                             ExecutiveLayer (goal state, GRU-based)
                             ConflictMonitor (ACC analog)
                             ThalamicGate (stateless, snapshot-based)

  PtolemyFace              — RENAME/EXPAND to PtolemyKernel
                             Add: thread pool management
                             Add: HippocampalBinder
                             Add: two-tier context buffer
                             Add: /DataInput bus

ADD NEW:
  SedenionLayer            — 16D hypercomplex multiplication
                             Load-bearing mathematical primitive
                             Test zero divisors in isolation FIRST
                             Known zero divisor pairs as unit tests

  HippocampalBinder        — Sedenion binding operation
                             Receives monad spectra as sedenions
                             Binding = sedenion multiplication
                             Zero divisors cancel interference
                             Constructive interference stratified:
                               Octonion subspace (dim 0-7 per unit)
                                 → CLASS_2 stable semantic core
                                 → Associative, consolidates
                               Sedenion shell (dim 8-15 per unit)
                                 → CLASS_3 experiential content
                                 → Non-associative, volatile
                                 → Does NOT consolidate

  CorpusIngester           — File-level input coordinator
                             Walks corpus files
                             Routes tokens to monads
                             Manages context stack
                             Required for Caesar corpus training

  ConsolidationBuffer      — Two-tier FIFO
    Tier 1: Working context (recent prompts, full sedenion products)
    Tier 2: Consolidated memory (octonion projections only)
    Trigger: FIFO eviction from Tier 1 → write octonion
             projection to Tier 2
    Thread safety: RLock per WordMonadNetwork instance
                   Read lock during inference
                   Write lock during training + consolidation

  ConfidenceScoring        — SEPARATE USER-DEFINED MODULE
                             See section below.

  ThreadJournal            — Per autonomous thread
                             Logs work state when parked
                             Enables resume vs restart

SEMANTIC_DIM RESTRUCTURING — MIGRATION CRITICAL
════════════════════════════════════════════════════════

CURRENT:  SEMANTIC_DIM = 512 (flat euclidean)
NEW:      SEMANTIC_DIM = 32 x 16 (32 sedenion units x 16 components)

THIS IS A BREAKING CHANGE ON ALL MONAD CHECKPOINTS.

DO THIS BEFORE ANY CORPUS ACQUISITION RUNS.
No checkpoints exist yet — this is the correct moment.

If checkpoints exist at time of change:
  Migration script required:
    1. Load each .pt monad checkpoint
    2. Project old 512-dim spectrum into 32x16 sedenion structure
    3. Validate zero divisor behavior on sample binding pairs
    4. Resave with new kernel version number

Kernel version number MUST be saved with every checkpoint.
Incompatible versions refuse to load without explicit
--migrate flag. Never silent compatibility failures.

════════════════════════════════════════════════════════
/DataInput BUS
════════════════════════════════════════════════════════

ALL input routes through /DataInput bus:
  - User prompts
  - Web retrieval results
  - File ingestion (corpus, whole files as input cycle unit)
  - Ptolemy's autonomous search results
  - Any future input source

ThalamicGate behavior on input:
  Close gate (pause input layer)
  Reconstruct (CharacterNeuron → monad activation → binding)
  Output
  Open gate

Gate is stateless between cycles.
Operates on FROZEN SNAPSHOT of candidate activations.
Does not hold reference to live monad pool.
(Avoids recursive list mutation during iteration.)

Web retrieval learning:
  Ptolemy learns from EVERY page retrieved.
  Page content enters /DataInput bus same as any input.
  Low confidence pages → quarantine buffer (not monad pool directly)
  Consolidation decides quarantine integration.
  This is why ConfidenceScoring is load-bearing, not optional.

════════════════════════════════════════════════════════
CONFIDENCE SCORING — SEPARATE MODULE (User-Defined)
════════════════════════════════════════════════════════

Truth/Propaganda confidence scoring.
Designed by user. Implemented as pluggable module.
Not part of ptolemy_core.py core architecture.
Registered with kernel as a scoring service.

Role in architecture:
  Gates learning from web input.
  High confidence → direct to monad pool via /DataInput
  Low confidence  → quarantine buffer
  Quarantine      → consolidation decides integration

Interface contract (to be defined by user):
  Input:  raw text + source metadata + sedenion binding product
  Output: confidence score (float 0.0-1.0)
          propaganda_flags (list)
          source_reliability (float)

Kernel treats this as a black box.
Any scoring methodology can be swapped in without
touching ptolemy_core.py.

EXECUTIVE LAYER TRAINING — BOOTSTRAP STRATEGY
════════════════════════════════════════════════════════

ExecutiveLayer holds goal state: research + self-teaching.
Two training signals:

Signal A (Novelty):
  Positive when output produces novel monad activation
  AND ConflictMonitor confidence was high.
  Requires monad pool to have sufficient entries.
  PHASE 2 activation.

Signal B (Corpus alignment):
  Distance between Ptolemy output spectral signature
  and Caesar/Cicero/Cato corpus signature.
  Available immediately.
  PHASE 1 bootstrap signal.

Phase 1 prototype uses Signal B only.
Signal A activates automatically once monad pool
has sufficient entries to measure novelty against.

KEY PRINCIPLES (load-bearing, not decorative)
════════════════════════════════════════════════════════

Rabies Principle:
  first_encountered in SemanticWord is permanently immutable.

Monad isolation:
  Each WordMonad trains only on its own word.
  Anti-forgetting by isolation, not just EWC.

Gate snapshot pattern:
  ThalamicGate operates on frozen candidate snapshot.
  Never holds reference to live monad pool.
  Prevents recursive mutation during competitive selection.

Sedenion stratification:
  CLASS_2 content → octonion subspace (associative, consolidates)
  CLASS_3 content → sedenion shell (non-associative, volatile)
  Zero divisors → structural noise cancellation (not learned)

Thread safety:
  RLock per WordMonadNetwork.
  Read lock: inference.
  Write lock: training, consolidation.
  Consolidation runs in background, never blocks inference.
  Kernel is single authority over all shared resources.

No silent failures:
  Incompatible checkpoint versions refuse to load.
  Require explicit --migrate flag.
  Kernel version saved with every checkpoint.
